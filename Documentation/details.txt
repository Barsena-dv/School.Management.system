# How to Read This ERD (Important Understanding)

### Identity Layer

* One user account ‚Üî one student profile
* Teachers/admins exist only in Users
* Students exist in both Users + Students

### Academic Layer

* Class ‚Üí Subjects ‚Üí Assessments ‚Üí Marks

### Activity Layer

* Attendance independent per subject per day
* Enrollment stores yearly history

### Financial Layer

* Student ‚Üí Fees ‚Üí Payments
  Never store payments directly on student (audit safe)

### System Layer

* Notifications ‚Üí communication
* Logs ‚Üí audit trail

# üìò SOFTWARE REQUIREMENTS SPECIFICATION (SRS)

## Student Academic Management Platform (MERN Based)

---

# 1. INTRODUCTION

## 1.1 Purpose

The purpose of this system is to develop a web-based academic management platform that enables an educational institution to manage students, teachers, attendance, assessments, fees, communication, and analytics through a role-based centralized application.

The system replaces manual record keeping and standalone tools with a real-time, multi-user platform supporting administrative decisions and academic monitoring.

---

## 1.2 Scope

The application provides:

* Multi-role login (Admin, Teacher, Student)
* Student lifecycle management
* Attendance monitoring
* Assessment and grading system
* Fees and payments tracking
* Notifications and announcements
* Reports and analytics dashboards
* Activity audit logging

The system will be accessible via browser and operate as a REST-based MERN application.

---

## 1.3 Definitions

| Term            | Meaning                                  |
| --------------- | ---------------------------------------- |
| RBAC            | Role Based Access Control                |
| Assessment      | Exam/quiz/assignment                     |
| At-Risk Student | Student with low attendance or marks     |
| Soft Delete     | Archive without removing database record |
| Academic Year   | Institutional session (YYYY-YYYY)        |

---

# 2. OVERALL DESCRIPTION

## 2.1 Product Perspective

The system is a standalone web application consisting of:

* React Frontend (Client)
* Node.js + Express Backend (API Server)
* MongoDB Database
* JWT Authentication Layer
* Event-Driven Services (notifications & analytics)

---

## 2.2 User Classes

| User    | Description                  |
| ------- | ---------------------------- |
| Admin   | Full system control          |
| Teacher | Academic operations          |
| Student | Self service academic access |

---

## 2.3 Operating Environment

| Layer      | Technology           |
| ---------- | -------------------- |
| Frontend   | React + Tailwind     |
| Backend    | Node.js + Express    |
| Database   | MongoDB              |
| Auth       | JWT + Refresh Tokens |
| Realtime   | WebSockets           |
| API Format | REST JSON            |

---

## 2.4 Constraints

* Must support concurrent users
* Must not permanently delete records
* Must validate all user inputs
* Must enforce role restrictions
* Must store hashed passwords only
